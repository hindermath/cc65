## iSAQB Architecture Planning Checks

- Plan updates to `docs/architecture/` for relevant architecture views:
  context, building blocks, runtime, deployment, quality scenarios, ADRs,
  risks, or technical debt.
- Plan ADRs for significant trade-offs, technology choices, module
  boundaries, interface contracts, or deployment decisions.
- Plan explicit quality-scenario validation where non-functional
  requirements drive the design.
- Plan architecture-risk review for unresolved constraints or accepted
  technical debt.
- If security-relevant architecture is affected, also plan the
  secure-architecture evidence from `architecture-governance`.

## Audit Evidence Planning

- Plan audit-ready Markdown evidence for this Spec-Kit run, including owner, reviewer, evidence path, and standard-specific applicability.
- Plan how each relevant checkpoint will be recorded as `Applicable`, `N/A`, or `Open`.
- Plan concrete evidence updates under the default evidence directory for this preset; do not leave checklist templates unfilled.
- Treat `Open` as temporary: assign an owner, follow-up, and re-evaluation trigger.
