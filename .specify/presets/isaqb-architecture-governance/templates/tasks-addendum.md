## iSAQB Architecture Tasks

- Add tasks to update relevant architecture views in `docs/architecture/`.
- Add tasks for ADRs where architecturally significant choices are made.
- Add tasks to define and validate quality scenarios.
- Add tasks to record architecture risks, accepted trade-offs, and
  technical debt.
- Add a final architecture review task for consistency between spec,
  plan, tasks, and architecture evidence.
- If security-relevant architecture is affected, add the corresponding
  secure-architecture tasks from `architecture-governance`.
